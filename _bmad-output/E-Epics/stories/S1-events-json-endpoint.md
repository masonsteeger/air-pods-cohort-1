# S1: `/events` JSON Endpoint

**Epic:** 01 — Public Event Calendar MVP
**Priority:** P1
**Estimate:** M
**Status:** Ready

---

## User Story

**As a** data consumer (visual calendar, future integrations)
**I want** a stable `/events` endpoint returning JSON
**So that** any client can fetch current event data

---

## Acceptance Criteria

- [ ] `GET /events` returns JSON array
- [ ] Each event object contains:
  - `groupName` (string) — Name of the meetup group
  - `eventTitle` (string) — Title of the specific event
  - `startDateTime` (ISO 8601) — Event start time with timezone
  - `status` (string) — `active` or `cancelled`
  - `links` (array) — Array of `{ label, url }` objects
- [ ] Events sorted chronologically (nearest first)
- [ ] Cancelled events included with `status: "cancelled"`
- [ ] Empty array returned when no events (not error)

---

## Technical Notes

- Endpoint serves pre-generated JSON (no runtime sheet access needed)
- JSON generated by sync pipeline (S2)
- Static file hosting is sufficient

---

## Schema

```json
[
  {
    "groupName": "DFW Pythoneers",
    "eventTitle": "April Meetup",
    "startDateTime": "2026-04-10T18:30:00-05:00",
    "status": "active",
    "links": [
      { "label": "Meetup", "url": "https://www.meetup.com/dfw-pythoneers/events/123456789/" }
    ]
  }
]
```
